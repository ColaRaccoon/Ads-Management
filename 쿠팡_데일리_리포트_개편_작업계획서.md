# 쿠팡 데일리 리포트 탭 전면 개편 작업계획서

> 문서 상태: **최종 확정 디자인 기준 구현 계획**
>
> 작성 목적: 다음 작업 세션에서 이 문서 하나만 읽고도 요구사항, 데이터 의미, API 계약, 화면 구조, 인쇄, 내보내기, 예외 처리, 테스트 및 완료 조건을 정확히 구현할 수 있도록 한다.
>
> 최종 디자인 원본: `design-previews/coupang-daily-report-mockup.html`
>
> 이 문서와 기존 코드가 충돌할 경우, **사용자가 확정한 화면 동작은 이 문서를 우선**하고 기존의 계산 정확성·불완전 계산 경고는 보존한다.

## 1. 최종 목표

현재 `/coupang/daily-report` 페이지의 범용 대형 테이블을 폐기하고, 아래 조건을 만족하는 쿠팡 전용 고밀도 데일리 리포트로 교체한다.

1. 옵션이 여러 개인 제품은 다음과 같이 한 묶음으로 표시한다.
   - 첫 번째 행: 제품 그룹 전체 합계
   - 다음 행들: 해당 그룹에 속한 각 옵션의 개별 실적
   - 그룹 행의 화살표로 옵션 행을 접거나 펼칠 수 있음
2. 옵션이 없는 단일 제품은 합계 행이나 빈 옵션 행 없이 한 행만 표시한다.
3. 화면과 인쇄물의 데이터 열은 다음 8개만 표시한다.
   - 제품/옵션
   - 쿠팡 원본매출
   - 원본 판매수량
   - 가구매수량
   - 광고비
   - 광고수익률
   - 오가닉 매출
   - 최종 순이익
4. 원본 판매수량, 광고비, 광고수익률, 최종 순이익에는 전일 값을 같은 셀 오른쪽에 작고 회색인 한 줄 텍스트로 표시한다.
5. 쿠팡 업로드 탭의 가구매수량 입력기에서 저장한 메모를 제품별 최하단 `기타사항` 행으로 표시한다.
6. 많은 상품을 한눈에 볼 수 있도록 모든 세로 여백, 행 높이, 카드 높이, 폰트 크기를 줄인다.
7. A4 가로 방향으로 최대한 많은 행을 출력할 수 있게 한다.

### 절대 변경하면 안 되는 확정 규칙

- 제품명 아래의 `옵션 상품`, `단일 상품`, `옵션 없음` 같은 보조 문구와 배지는 **표시하지 않는다**.
- 옵션명 옆에도 `옵션` 같은 추가 라벨을 **표시하지 않는다**.
- 전일 값 때문에 별도 행을 만들거나 셀을 두 줄로 늘리지 않는다.
- `기타사항`은 독립된 열이 아니라 해당 제품 묶음의 최하단을 가로지르는 `colSpan={8}` 행이다.
- **유효한 메모가 하나도 없으면 `기타사항` 행의 `<tr>` 자체를 렌더링하지 않는다.**
- 메모가 없을 때 빈 셀, 빈 행, 빈 테두리, 빈 여백, `기타사항 없음` 문구, 높이 예약을 모두 금지한다.
- 화면용 그룹 합계와 옵션별 값은 같은 계산 원천을 사용해야 하며 프런트엔드가 금액을 재계산하지 않는다.

## 2. 현재 코드에서 확인된 기반

### 현재 화면

- 파일: `apps/web/src/app/coupang/daily-report/page.tsx`
- 현재는 `DataTable`을 사용하며 제품/그룹 보기 선택과 30개 이상의 열을 표시한다.
- 현재 요청은 단순 열 축소가 아니라, 그룹 행·자식 옵션 행·기타사항 행을 함께 렌더링해야 하므로 `DataTable` 재사용이 적합하지 않다.
- 이 페이지를 전용 `<table>` 렌더링으로 교체한다.

### 현재 API

- 컨트롤러: `apps/api/src/coupang/coupang.controller.ts`
- 서비스: `apps/api/src/coupang/coupang.service.ts`
- 엔드포인트: `GET /coupang/daily-report?date=YYYY-MM-DD&groupBy=...`
- 현재 `dailyReport()`는 하루치 `buildProductProfitRows()` 결과를 만들고, 선택된 `groupBy`에 따라 제품 또는 그룹만 반환한다.
- 기존 그룹 응답은 자식 옵션을 의도적으로 제거한 평면 응답이므로, 최종 디자인을 위해 계층 응답으로 변경해야 한다.

### 이미 사용할 수 있는 계산 및 그룹 기반

- `ProductProfitRow`
- `buildProductProfitRows()`
- `aggregateCoupangProductProfitRowsByGroup()`
- `summarizeCoupangProductProfitRows()`
- `hasDailyReportActivity()`

기존 계산 함수는 가능한 한 유지한다. 특히 가구매 조정, 비용, ROAS, 오가닉 매출, 최종 순이익의 계산식을 프런트엔드에서 복제하지 않는다.

### 메모 저장 기반

- DB 모델: `CoupangManualPurchase.memo String?`
- 유일 키: `(purchaseDate, coupangProductId)`
- 업로드 화면은 이미 `quantity`와 `memo`를 함께 저장한다.
- 저장 성공 후 `["coupang-daily-report"]` 쿼리를 이미 무효화한다.
- 따라서 이 개편을 위한 Prisma 스키마 변경이나 DB 마이그레이션은 필요하지 않다.

## 3. 데이터 열의 정확한 의미

화면, 인쇄, CSV, XLSX에서 동일한 계산 의미를 사용한다.

| 화면 열 | API 원천 | 정의 및 표시 규칙 |
|---|---|---|
| 쿠팡 원본매출 | `reportedSalesKrw` | 쿠팡 원본 파일 기준 총매출. 원 단위 정수로 반올림하여 표시 |
| 원본 판매수량 | `reportedSalesQuantity` | 쿠팡 원본 파일 기준 판매수량. `개` 단위 |
| 가구매수량 | `manualPurchaseQuantity` | 선택 날짜에 저장된 가구매수량 합계. `개` 단위 |
| 광고비 | `adSpendKrw` | 선택 날짜의 광고비 합계. 원 단위 |
| 광고수익률 | `roas` | `광고 전환매출 ÷ 광고비`. API 값은 비율이며 화면에서 `× 100` 후 `%` 표시 |
| 오가닉 매출 | `actualOrganicSalesKrw` | `actualNetSalesKrw - adConversionSalesKrw`. 가구매 조정이 반영된 실제 순매출에서 광고 전환매출을 차감한 값 |
| 최종 순이익 | `marginKrw` | 기존 쿠팡 수익 계산의 최종 합산 순이익 |

### 숫자 표시

- 금액: `1,234,000원`
- 수량: `1,234개`
- ROAS: 소수점 한 자리, 예: `461.0%`
- 양수 순이익: `+353,800원`
- 음수 순이익: `-42,100원`
- 값이 계산 불가인 `null`: `-`
- 정상적인 0: `0원`, `0개`
- ROAS는 광고비가 0이면 `null`이며 `-`로 표시한다. 무한대나 `0.0%`로 바꾸지 않는다.
- 오가닉 매출이 음수여도 값을 숨기지 않고 음수로 표시하며 기존 경고를 유지한다.

### 불완전 계산

- 개별 상품이나 그룹의 `marginKrw`가 `null`이면 `-`로 표시한다.
- 그룹 내 하나라도 순이익 계산이 불완전하면 그룹 최종 순이익도 확정값처럼 합산하지 않고 `-`로 표시한다.
- 전체 요약이 불완전한 경우 기존 `knownMarginKrw`를 부분 합계로 표시할 수 있으나, 반드시 짧은 경고 스트립으로 “일부 상품이 제외된 부분 합계”임을 알린다.
- 경고/계산상태를 별도 열로 추가하지 않는다. 데이터 열은 확정된 8개를 유지한다.

## 4. 전일 비교 규칙

선택일을 `D`라고 하면 전일은 UTC 날짜 전용 연산으로 계산한 `D - 1일`이다.

- API의 기존 `previousDate(date: Date)`와 `formatDateOnly()`를 사용한다.
- 브라우저 로컬 시간이나 `new Date("YYYY-MM-DD")`의 타임존 보정에 의존하지 않는다.
- 전일 비교 대상은 아래 네 항목만이다.
  - 원본 판매수량
  - 광고비
  - 광고수익률
  - 최종 순이익
- 쿠팡 원본매출, 가구매수량, 오가닉 매출에는 전일 보조 텍스트를 표시하지 않는다.

표현 예:

- `46개 (전일 42개)`
- `296,000원 (전일 281,000원)`
- `447.6% (전일 431.2%)`
- `+356,900원 (전일 +329,600원)`

구현 시 현재 값과 전일 값은 같은 `.metric-inline` 안에 두고, 전일 값은 `<small>`로 렌더링한다. `<small>`은 오른쪽에 붙으며 줄바꿈을 허용하지 않는다.

전일 데이터가 없는 경우:

- 수량: `(전일 0개)`
- 광고비: `(전일 0원)`
- 광고수익률: `(전일 -)`
- 최종 순이익: `(전일 -)`  
  단, 전일 행이 존재하고 확정 순이익이 정확히 0이면 `(전일 0원)`이다.

전일에만 실적이 있고 선택일에 실적이 전혀 없는 단일 상품은 선택일 리포트에 새 최상위 행으로 만들지 않는다. 다만 오늘 표시되는 제품 그룹에 속한 옵션이라면 아래의 “모든 옵션 표시 규칙”에 의해 자식 옵션 행으로 볼 수 있다.

## 5. 최종 API 응답 계약

`groupBy` 선택 UI는 제거한다. 데일리 리포트는 항상 그룹 합계와 옵션을 함께 제공하는 단일 계층 구조이다.

권장 TypeScript 계약:

```ts
export type CoupangDailyPreviousMetrics = {
  reportedSalesQuantity: number;
  adSpendKrw: number;
  roas: number | null;
  marginKrw: number | null;
};

export type CoupangDailyVisibleMetrics = {
  reportedSalesKrw: number;
  reportedSalesQuantity: number;
  manualPurchaseQuantity: number;
  adSpendKrw: number;
  roas: number | null;
  organicSalesKrw: number | null;
  marginKrw: number | null;
};

export type CoupangDailyProductRow = CoupangDailyVisibleMetrics & {
  rowType: "PRODUCT";
  productId: string;
  productName: string;
  groupId: string | null;
  groupName: string | null;
  memo: string | null;
  previous: CoupangDailyPreviousMetrics;
  calculationStatus: "COMPLETE" | "INCOMPLETE";
  warnings: string[];
};

export type CoupangDailyGroupRow = CoupangDailyVisibleMetrics & {
  rowType: "GROUP";
  groupId: string;
  groupName: string;
  productName: string;
  childProductCount: number;
  children: CoupangDailyProductRow[];
  previous: CoupangDailyPreviousMetrics;
  calculationStatus: "COMPLETE" | "INCOMPLETE";
  warnings: string[];
};

export type CoupangDailyReportRow =
  | CoupangDailyGroupRow
  | CoupangDailyProductRow;

export type CoupangDailySummary = CoupangDailyVisibleMetrics & {
  isComplete: boolean;
  knownMarginKrw: number;
  incompleteProductCount: number;
  excludedNetSalesKrw: number;
  excludedSalesQuantity: number;
};

export type CoupangDailyReportResponse = {
  date: string;
  previousDate: string;
  summary: {
    current: CoupangDailySummary;
    previous: CoupangDailySummary;
  };
  rows: CoupangDailyReportRow[];
};
```

### 계약 관련 결정 사항

- 기존 `CoupangDailyReportResponse.groupBy`는 제거한다.
- `GET /coupang/daily-report`는 `date`만 사용한다.
- 과거 북마크에 `groupBy` 쿼리가 남아 있어도 서버는 불필요한 쿼리 파라미터로 무시할 수 있어야 한다.
- 현재 값은 기존 필드명을 최대한 유지하고, 전일 비교만 `previous` 객체에 둔다.
- `organicSalesKrw` 응답에는 반드시 기존의 `actualOrganicSalesKrw` 값을 투영한다. `reportedOrganicSalesKrw`를 넣으면 안 된다.
- 메모는 상품/옵션별 `memo` 한 개로 내려준다. 그룹의 기타사항은 프런트엔드가 `children`의 유효 메모를 표시 순서대로 모아 한 줄로 렌더링한다.
- API는 공백 메모를 `null`로 정규화한다.

## 6. API 집계 및 계층 생성 알고리즘

`apps/api/src/coupang/coupang.service.ts`의 `dailyReport()`를 다음 순서로 재구성한다.

### 6.1 날짜 및 원천 데이터

1. `date` 누락 시 기존 `DATE_REQUIRED` 오류를 유지한다.
2. 날짜 형식 오류 시 기존 `INVALID_DATE` 오류를 유지한다.
3. `previousDate(date)`와 `formatDateOnly()`로 전일 문자열을 만든다.
4. 아래 데이터를 병렬 조회한다.
   - 선택일 `buildProductProfitRows()`
   - 전일 `buildProductProfitRows()`
   - `CoupangProduct` 전체와 `group` 관계
   - 선택일 `CoupangManualPurchase`의 `coupangProductId`, `productDisplayName`, `memo`
5. 전체 요약은 그룹 집계 전의 제품 행으로 각각 계산한다.
   - 현재: `summarizeCoupangProductProfitRows(currentProductRows)`
   - 전일: `summarizeCoupangProductProfitRows(previousProductRows)`
   - 그룹 합계와 옵션을 동시에 합산하여 이중 계산하면 안 된다.

### 6.2 메모 정규화

```ts
function normalizeDailyMemo(value: string | null | undefined) {
  const trimmed = value?.trim() ?? "";
  return trimmed.length > 0 ? trimmed : null;
}
```

- `null`, `undefined`, `""`, `"   "`, 탭/줄바꿈만 있는 값은 모두 `null`.
- 내용 앞뒤 공백만 제거한다.
- 중간의 사용자 문구를 임의로 수정하지 않는다.
- React 렌더링을 사용하여 HTML을 이스케이프하고 `dangerouslySetInnerHTML`을 사용하지 않는다.

### 6.3 표시 대상 결정

- 최상위 표시 대상은 선택일의 `hasDailyReportActivity(row) === true`인 제품으로 결정한다.
- 전일에만 실적이 있는 미분류 단일 상품은 오늘의 최상위 행으로 추가하지 않는다.
- 현재 활동 제품이 그룹에 속하면 그룹 단위로 하나의 최상위 행을 만든다.
- 그룹에 속하지 않으면 단일 상품 행을 만든다.
- 비활성 상품이라도 선택일에 실제 데이터가 있으면 누락하지 않는다.

### 6.4 그룹의 모든 옵션 표시

사용자의 “옵션을 모두 취합하고 그 아래 각 옵션별 데이터 표시” 요구를 다음처럼 해석하고 고정한다.

- 어떤 그룹이 선택일 표시 대상이 되면 그 그룹의 **활성 상품/옵션 전체**를 자식으로 표시한다.
- 선택일 실적이 없는 활성 옵션도 0 값으로 자식 행에 표시한다.
- 선택일 또는 전일 데이터가 있는 비활성 옵션도 역사 데이터 누락 방지를 위해 포함한다.
- 선택일과 전일 데이터가 모두 없고 비활성인 옵션은 표시하지 않는다.
- 데이터가 없는 자식 옵션의 기본값:
  - 합산형 수치: `0`
  - ROAS: `null`
  - 오가닉 매출: `0`
  - 최종 순이익: `0`
  - 계산상태: `COMPLETE`
  - 경고: `[]`
- 그룹 부모의 현재 합계는 실제 `currentProductRows`를 기존 그룹 집계 함수로 합산한다. 화면에 보이기 위해 만든 0 행을 계산 원천에 다시 넣지 않는다.
- 그룹 부모의 전일 값은 해당 그룹의 모든 전일 제품 행을 집계하여 만든다.
- 그룹 ROAS는 옵션별 ROAS 평균이 아니라 `그룹 광고 전환매출 합계 ÷ 그룹 광고비 합계`이다.
- 그룹 오가닉 매출과 순이익은 기존 `strictSumNullable` 의미를 보존한다.
- `childProductCount`는 실제 화면에 포함한 자식 옵션 수로 덮어쓴다.

### 6.5 정렬

- 최상위 그룹/단일 상품: 선택일 `reportedSalesKrw` 내림차순, 동률이면 표시명 오름차순.
- 그룹 내부 옵션: 선택일 `reportedSalesKrw` 내림차순, 동률이면 `CoupangProduct.sortOrder`, 다시 동률이면 표시명 오름차순.
- 메모 조각의 순서는 화면의 자식 옵션 순서와 같게 한다.

### 6.6 DTO 투영

서비스 내부의 거대한 `ProductProfitRow`를 그대로 노출하지 말고 `toDailyProductRow()`와 `toDailyGroupRow()` 같은 전용 순수 함수를 만든다.

- 화면에 필요한 7개 수치만 현재 DTO에 투영한다.
- `organicSalesKrw`에는 `actualOrganicSalesKrw`를 넣는다.
- `previous`에는 지정된 4개 수치만 넣는다.
- 현재/전일 매칭 키:
  - 상품: `productId`
  - 그룹: `groupId`
- 이름 문자열로 매칭하지 않는다.

## 8. 기타사항 행의 최종 동작

이 항목은 최종 승인에서 특별히 확정된 필수 조건이다.

### 그룹 제품

1. 펼쳐진 옵션 행들을 모두 렌더링한다.
2. `children` 중 `memo !== null`인 옵션만 모은다.
3. 유효 메모가 하나 이상이면 마지막 옵션 바로 다음에 기타사항 행 하나를 렌더링한다.
4. 표현:

```text
기타사항  블랙 리뷰 보강 · 베이지 재고 확인
```

- `기타사항`: 회색 굵은 라벨
- 옵션명: 브랜드 색상의 작은 굵은 텍스트
- 메모 사이: `·`
- `td colSpan={8}`

### 단일 제품

- `memo !== null`이면 제품 행 바로 다음에:

```text
기타사항  신규 상품 리뷰 보강
```

- 단일 제품에서는 제품명을 메모 앞에 다시 반복하지 않는다.

### 메모가 없는 경우

아래 조건을 코드와 테스트에서 모두 강제한다.

```tsx
const notes = children.filter((child) => child.memo !== null);

return notes.length > 0 ? (
  <tr className="coupang-daily-memo-row">...</tr>
) : null;
```

단일 제품:

```tsx
{row.memo !== null ? <MemoRow memo={row.memo} /> : null}
```

금지 예:

```tsx
<tr><td colSpan={8}></td></tr>
<tr><td colSpan={8}>기타사항 없음</td></tr>
<div className="memo-space" />
```

즉, 메모가 없으면 DOM에 기타사항 행이 없어야 하며 다음 제품 행이 즉시 이어진다.

### 접기, 검색, 인쇄와의 관계

- 그룹을 접으면 옵션 행과 해당 그룹 기타사항 행을 함께 숨긴다.
- 인쇄에서는 화면에서 접혀 있어도 모든 옵션과 존재하는 기타사항을 강제로 표시한다.
- 인쇄 시에도 메모가 없는 제품의 기타사항 행을 새로 만들지 않는다.
- 검색어가 메모와 일치하면 부모 그룹/단일 제품을 검색 결과에 포함한다.
- 그룹 메모 검색 시 해당 메모를 가진 옵션과 부모 그룹을 표시한다.

## 9. 접기/펼치기 및 검색 상태

페이지 상태:

```ts
const [collapsedGroupIds, setCollapsedGroupIds] = useState<Set<string>>(new Set());
const [searchQuery, setSearchQuery] = useState("");
```

- 기본 상태는 모든 그룹 펼침.
- 개별 그룹 토글은 `groupId`만 Set에 추가/제거한다.
- `모두 접기`는 현재 응답의 모든 그룹 ID를 Set에 넣는다.
- `모두 펼치기`는 Set을 비운다.
- 날짜가 바뀌어 새 응답을 받으면 존재하지 않는 그룹 ID를 Set에서 제거한다.

검색은 DOM의 `style.display`를 직접 조작하지 말고 순수 헬퍼와 `useMemo`로 계산한다.

검색 기준:

- 그룹명
- 단일 제품명
- 옵션명
- 메모
- 대소문자 구분 없음
- 검색어 앞뒤 공백 제거

검색 동작:

- 그룹명이 일치하면 그룹과 모든 자식 옵션을 표시한다.
- 특정 옵션명 또는 메모만 일치하면 부모 그룹과 일치하는 옵션만 표시한다.
- 검색 중 일치 그룹은 임시로 펼쳐 보이지만 `collapsedGroupIds` 자체를 바꾸지 않는다.
- 검색어를 지우면 검색 전 사용자의 접힘 상태로 돌아간다.
- 결과가 없으면 테이블 헤더 아래에 작은 빈 상태를 표시한다.

## 11. 인쇄 사양

A4 가로 한 장에 최대한 많은 제품을 담는 것이 우선이다.

### CSS

```css
@media print {
  @page coupang-daily-report {
    size: A4 landscape;
    margin: 6mm;
  }
}
```

페이지 루트에:

```css
.coupang-daily-report-page {
  page: coupang-daily-report;
}
```

### 인쇄 시 숨길 요소

- 사이드바
- 상단 앱 바
- 날짜 입력
- 인쇄/XLSX/CSV 버튼
- 검색 입력
- 모두 접기/펼치기 버튼
- 화면 전용 안내 문구

### 인쇄 시 강제할 요소

- 배경 흰색, 글자 검정에 가까운 색
- 카드 그림자, 과도한 둥근 모서리 제거
- 페이지 여백 0
- 테이블 `min-width: 0`, `width: 100%`
- sticky 해제
- 테이블 약 7pt, 전일 약 5.5~6pt
- 셀 패딩 약 2~3px
- `thead { display: table-header-group; }`
- 각 제품 `<tbody>`에 `break-inside: avoid` 및 `page-break-inside: avoid`
- 접혀 있는 옵션 및 존재하는 메모 행 강제 표시

중요: 접힌 행을 React에서 아예 제거하면 CSS가 인쇄 시 되살릴 수 없다. 따라서 그룹 옵션/메모는 DOM에 유지하고 화면에서는 클래스 또는 `hidden` 스타일로 숨기는 방식으로 구현한다. 단, **원래 메모가 없는 경우의 메모 행은 처음부터 DOM에 만들지 않는다.**

인쇄 버튼은 `window.print()`를 호출한다.

### 인쇄 검증

- Chromium 인쇄 미리보기에서 A4 가로, 배율 100% 기준 확인
- 오른쪽 열 잘림 없음
- 첫 열이 겹치지 않음
- 헤더가 다음 페이지에 반복됨
- 제품 그룹과 첫 옵션 사이 불필요한 페이지 분리 최소화
- 메모 없는 제품 사이에 빈 줄 없음
- 화면에서 접은 그룹도 인쇄물에는 모든 옵션이 보임

## 12. CSV/XLSX 내보내기

현재 제공 중인 CSV/XLSX 기능은 유지한다.

### 화면과 내보내기의 관계

- 화면/인쇄에는 확정된 8개 열만 보인다.
- 내보내기에는 이중 합산 방지를 위한 `행구분`과 기계 판독 가능한 전일 열을 추가할 수 있다.
- 권장 열 순서:
  1. 행구분
  2. 제품/옵션
  3. 쿠팡 원본매출
  4. 원본 판매수량
  5. 전일 원본 판매수량
  6. 가구매수량
  7. 광고비
  8. 전일 광고비
  9. 광고수익률
  10. 전일 광고수익률
  11. 오가닉 매출
  12. 최종 순이익
  13. 전일 최종 순이익

`행구분` 값:

- `전체합계`
- `그룹합계`
- `옵션`
- `단일제품`
- `기타사항`

### 계층 평탄화 순서

1. 전체 합계
2. 그룹 합계
3. 그룹의 옵션들
4. 유효 메모가 있으면 기타사항
5. 다음 그룹 또는 단일 제품

금액/수량/ROAS는 XLSX에서 문자열이 아니라 숫자로 유지하고 기존 셀 스타일을 사용한다. CSV는 기존 BOM 처리 유틸리티를 유지한다.

기타사항 행:

- 제품/옵션 열에 `기타사항: 블랙 리뷰 보강 · 베이지 재고 확인`
- 숫자 셀은 빈 값
- 메모가 하나도 없으면 기타사항 내보내기 행도 생성하지 않는다.

그룹 합계와 옵션 행이 함께 있으므로 `행구분`을 생략하지 않는다. 사용자가 합계를 낼 때 `그룹합계`와 `옵션`을 동시에 더하지 않도록 하기 위한 안전장치이다.

## 15. 테스트 계획

### 15.1 API 테스트

최소 다음을 검증한다.

1. `date` 누락 및 잘못된 날짜 오류 유지
2. 현재 제품과 전일 제품을 `productId`로 정확히 매칭
3. 전일 데이터가 없을 때 수량/광고비 0, ROAS/순이익 `null`
4. 그룹 부모에 `children`이 포함됨
5. 그룹 현재 합계가 현재 자식 실적 합과 일치
6. 그룹 전일 비교가 전일 그룹 합계와 일치
7. 그룹 ROAS가 옵션 ROAS 단순 평균이 아닌 가중 집계
8. 오늘 활동이 있는 그룹은 오늘 0인 활성 옵션도 자식으로 포함
9. 전일에만 데이터가 있는 활성 그룹 옵션도 오늘 표시되는 그룹의 자식으로 포함
10. 전일에만 활동한 미분류 단일 제품은 오늘 최상위 행으로 만들지 않음
11. 그룹 없는 상품은 단일 `PRODUCT` 행
12. 메모가 옵션별로 올바르게 연결됨
13. `null`, 빈 문자열, 공백뿐인 메모가 `null`로 반환됨
14. `actualOrganicSalesKrw`가 `organicSalesKrw`로 투영됨
15. 불완전 옵션이 있는 그룹의 `marginKrw`가 `null`
16. 요약은 그룹과 옵션을 중복 합산하지 않음
17. 비활성 상품이라도 선택일 데이터가 있으면 누락하지 않음

### 15.2 프런트엔드 헬퍼 테스트

1. 빈 검색어는 원래 순서와 모든 행을 유지
2. 그룹명 검색은 그룹과 전체 옵션을 표시
3. 옵션명 검색은 부모와 일치 옵션만 표시
4. 메모 검색은 부모와 해당 옵션/단일 제품을 표시
5. 검색 종료 후 접힘 상태 복원에 필요한 상태를 변경하지 않음
6. 그룹 메모 조각이 옵션 표시 순서와 같음
7. `memo: null`, `""`, `"   "`이면 `dailyRowNotes()`가 빈 배열
8. 메모가 없으면 내보내기 `기타사항` 행이 없음
9. 메모가 있으면 기타사항 행이 정확히 한 번 생성
10. 계층 내보내기 순서가 그룹합계 → 옵션 → 기타사항
11. 현재/전일 내보내기 숫자 값이 문자열로 변환되지 않음
12. 돈·수량·ROAS·음수/0/null 표시

### 15.3 수동 화면 검증

- 옵션 그룹 3개 이상
- 단일 상품 5개 이상
- 메모 있는 그룹
- 메모 없는 그룹
- 메모 있는 단일 제품
- 메모 없는 단일 제품
- 오늘 0, 전일 실적만 있는 옵션
- 광고비 0
- 음수 오가닉 매출
- 음수 순이익
- 불완전 계산 상품
- 긴 제품명/옵션명/메모
- 검색 결과 없음

가장 중요한 육안 확인:

- 메모 없는 제품 다음에는 즉시 다음 제품 행이 붙어 있어야 한다.
- “옵션 상품”, “단일 상품”, “옵션 없음” 문구가 화면 어디에도 없어야 한다.
- 전일 값은 현재 값 오른쪽에 있고 행 높이를 늘리지 않아야 한다.

## 19. 작업 중 금지 사항

- 화면에서 그룹 합계나 순이익을 다시 계산하지 않는다.
- 상품명 문자열로 현재/전일 또는 그룹을 연결하지 않는다.
- 옵션과 그룹 합계를 전체 요약에 동시에 더하지 않는다.
- `groupBy` 셀렉트를 남겨 두지 않는다.
- 확정된 8개 열 외에 상태, 비용 상세, 상품 종류 열을 화면에 추가하지 않는다.
- 전일 값을 별도 행이나 별도 열로 화면에 추가하지 않는다.
- 메모 없는 상품에 기타사항 자리만 남기지 않는다.
- 인쇄를 위해 가짜 메모 행을 만들지 않는다.
- 기존 가구매 계산식을 이번 UI 작업에서 변경하지 않는다.
- 이 작업을 이유로 Prisma 마이그레이션을 만들지 않는다.
- 최종 디자인 원본 HTML의 가짜 데이터를 실제 앱에 하드코딩하지 않는다.
